项目研究
主要内容
研究内容：
需求分析：对校园师生的信息需求和交流需求进行深入调研和分析，利用
数字人技术实现校园资源的智能调度和管理，明确智能问答系统在校园教育中
的应用场景和功能要求。
模型训练数据准备与处理：收集和整理校园教育相关的数据，包括但不仅
限于学校历年的分数线、校园规章制度、学生评定手册等等，多维度调研收集
与学校相关的精准数据，并进行数据清洗和预处理，以便于后续的模型训练和
优化。
模型的选择：根据需求分析的结果，用 DeepSeek 选择合适的自然语言处理
模型作为基础模型，对基础模型进行模型训练。
Agent 框架设计：利用 Agent 技术实现任务的划分、规划和执行，形成了
定制化的任务处理流程。代理负责接收用户输入，分析意图，并协调大模型和
知识库生成模块回答。
数字人集成：集成 NLP 模型，支持自然对话，集成 ASR 和 TTS 模块，实现
语音输入输出，通过情感分析调整数字人的反应。
基础模型的后训练：为了让基础的模型能够解答学生以及教师的问题，需
要向其投喂上述准备好的模型训练的无标注的数据。通过在大量数据中进行有
监督或无监督的训练，使模型具备较强的语义理解和问题解答能力。
模型微调：为了让模型在预训练后能够对用户的问题回复更加准确，需要
再为其投喂学习经过人工标注后的数据，通过学习标注后的数据，模型能够大
大提升回复的质量（如图 1-1）。

微调技术（Fine-tuning）：通过对预训练的大型语言模型（如 DeepSeek）
进行初始化，设置适应校园场景的特定需求。参数过程中，利用校园相关的训
练数据，例如校园政策、课程安排和生活服务等领域的问答，来调整模型参数。
这种方法能够提升模型在校园场景中的回答准确性和专业性。
训练数据思维链（Chain of Thought）：构建思维链数据，使用标注好的
思维链数据，对大模型（如 DeepSeek）进行强度，增强其多步推理能力。通过
迭代训练，优化模型在复杂问题上的表现。提升模型对用户查询的理解能力和
回答的精准性，并非简单回答，而是能够逐步引导用户解决问题。
搜索增强生成技术（RAG）：在模型训练和推理过程中，结合校园专属知识
库，通过检索相关信息来生成答案。RAG 技术使模型能够从实时更新的知识库
中获取最新数据，从而保证答案的时效性和准确性，同时减少模型生成不准确
或推理内容的可能性。
集成到数字人系统中：将用户输入（文字或语音转文字）发送给 API，获
取答案后触发数字人的语音输出或动画。通过用户反馈和测试结果进行迭代改
进，确保系统的稳定性、可用性和良好的用户体验（如图 1-5）。

（1）研究目的
研究意义与目的：
本项目的研究意义主要体现在以下几个方面:
第一，虚拟数字人作为人工智能与新媒体技术融合的产物，已成为技术实践与理
论探索的重要对象[1]。通过利用商用大模型技术开发适用于本校的智慧机器人问答
[2]系统，可以为师生提供更便捷、高效的信息获取和交流方式，提升教学和学习效
率，也能够推动人工智能技术在校园场景下的落地应用，具有重要的意义[3]。
第二，结合 Agent 技术（任务分解与调度）、大模型（语义理解与生成）与知识
库（领域知识增强），突破单一技术的局限性，提升系统综合性能。通过加入类人的
形象、肢体动作等信息，获得更加自然、舒适的交互体验，其具备一定的自然语言能
力，输入文本后，数字人能够以真人嘴唇吻合形式进行文本播报[4]，使系统服务更
贴近真实人际沟通，提升用户信任感与接受度。
第三，接入国内最新的商用大模型（DeepSeek 大模型），对大模型进行微调和
优化，深度思考人工智能技术在校园教育管理、学术交流等方面的创新应用模式，为
智能教育、智慧校园的建设做出积极贡献。同时，该项目可为学生提供人工智能思维
构建与实践的机会，增强学生的创新意识，提升学生的团队协作能力，更好地掌握
人工智能前沿技术[5]。
（2）研究内容
研究内容：
需求分析：对校园师生的信息需求和交流需求进行深入调研和分析，利用数字人
技术实现校园资源的智能调度和管理，明确智能问答系统在校园教育中的应用场景和
功能要求。
模型训练数据准备与处理：收集和整理校园教育相关的数据，包括但不仅限于学
校历年的分数线、校园规章制度、学生评定手册等等，多维度调研收集与学校相关的
精准数据，并进行数据清洗和预处理，以便于后续的模型训练和优化。
模型的选择：根据需求分析的结果，用 DeepSeek 选择合适的自然语言处理模型
作为基础模型，对基础模型进行模型训练。
Agent 框架设计：利用 Agent 技术实现任务的划分、规划和执行，形成了定制化

（3）国、内外研究现状和发展动态
同类产品研究现状与存在的问题：
通过全面调查，国内多数高校已部署基础问答系统，例如清华大学“小木”机
器人、复旦大学智能教务系统；国外的像韩国 KAIST、日本东京大学都在尝试将虚拟
数字人与问答系统结合，然而，这些系统无法解决垂直领域的问题。同时，近年逐
步引入预训练大模型（如百度 PLATO、智谱 ChatGLM），还有美国多所大学采用 AI
助手（如 Georgia Tech 的“Jill Watson”）辅助教学管理，但在校园场景的适配
性仍需优化，场景覆盖较窄。综上所述，通过收集高质量的业务数据对大模型进行
微调，可实现垂直业务的问答[6]。所以根据团队目前的调查，发现现有系统多以文
本为主，数字人的语音、表情等拟人化交互技术成熟度低。同时，对于校园特定需
求的定制化产品相对较少，而且对于学生的业务问题[7]，例如教学、科研和生活问
题，目前学校只能通过每个院系各班的信息员去追问跨部门的老师，然后信息员再

（4）创新点与项目特色
项目结合国内最新可商用大模型（DeepSeek 大模型）在技术完备性上的优势，
充分挖掘校园教育场景下的智能问答系统需求，开展深入研究和探索，为校园教育
提供更加智能化、个性化的服务，展示了智能问答系统的工作流程。但是，现有的
DeepSeek 大模型是属于通用大模型，在语义理解、知识库构建、用户交互等方面还

（5）技术路线、拟解决的问题及预期成果
1. 数据获取与清洗问题：通过查阅、询问、问卷调查等方式收集整理尽可能多
地关于学校的问题，构建出足够详细的数据集，然后对数据进行清洗，确保数据的
准确性和可用性。
2. 模型思维链构建问题：系统用户理解复杂的解释并需要逻辑清晰的回答，尤

其是在回答需要多步推理的习得相关问题时（例如“如何办理转专业手续？”），

流程等。通过关键问题解决的推理步骤，模型训练具备逻辑推理能力。
3. 解决大模型幻觉问题：运用检索增强生成（RAG）技术，构建一个校园专属
知识库，将学校的相关数据（政策、通知、流程等）整理成格式化的内容。在回答
时，模型先从知识库搜索相关信息，再根据搜索结果生成回答。通过 RAG 技术，确
保回答基于真实数据，显着减少幻觉现象，提高回答的可靠性和时效性。
4. 如何让数据更加有效地影响大模型回复的准确性：为了大模型的准确性，需

精准。

下文，负责协调大模型和知识库的交互。通过多轮对话设计，确保系统能够根据用
户的历史提问提供连贯的答案。编写代码处理用户输入（语音或文字），调用 API
并触发动画，增强用户体验感。
