# 数字人视频素材规范

将数字人视频放在本目录，文件名固定如下：

- `greeting.mp4`：欢迎开场（一次性播放）
- `idle.mp4`：待机状态（循环播放）
- `teaching.mp4`：讲解状态（循环播放）

## 推荐编码规范

- 封装格式：`MP4`
- 视频编码：`H.264`（High Profile）
- 音频编码：`AAC-LC`
- 帧率：`25` 或 `30` fps（全素材保持一致）
- 分辨率：建议统一（如 `1080x1920` 或 `720x1280`）
- 码率：建议 `2Mbps-8Mbps`（过高会影响加载稳定性）

## 建议时长

- `greeting.mp4`：5-12 秒
- `idle.mp4`：6-20 秒（适合循环）
- `teaching.mp4`：6-20 秒（适合循环）

## 常见问题

- 页面显示“预留占位”：
  - 先检查文件名是否完全匹配（区分大小写）。
  - 检查视频编码是否为 H.264 + AAC。
- 欢迎视频无声：
  - 浏览器可能拦截有声自动播放，属于浏览器策略限制。

## 调试模式

在 `.env` 中设置：

```env
VITE_VIDEO_DEBUG=true
```

页面会显示当前 `cue/src/errorCode/readyState`，便于排查。
