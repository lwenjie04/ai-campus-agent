# 校园智能问答平台介绍文档

## 一、项目背景
高校信息通常分散在教务处、学生工作处、学院通知、附件文件和各类公告页面中。学生在查询奖学金、转专业、课程认定、补退选、宿舍服务等信息时，往往需要多次搜索和比对，信息获取成本较高。

本项目围绕“校园服务智能化”这一目标，构建了一个集智能问答、知识检索、数字人展示、学生社区互动于一体的校园智能服务平台。用户可以通过自然语言提出问题，系统在校园知识库和社区知识的辅助下生成结构化回答，并提供来源依据，提升信息获取效率和可信度。

## 二、项目目标
本项目的目标包括：

- 提供统一的校园信息问答入口
- 利用 RAG 技术提升问答准确性和可追溯性
- 通过数字人提升交互体验和展示效果
- 构建学生社区，让经验型知识能够沉淀和复用
- 建立管理员审核与知识入库流程，保证知识质量

## 三、系统定位
本系统不是通用聊天机器人，而是面向高校场景的垂直智能服务平台。它更关注：

- 校园问答的准确性
- 信息来源的真实性
- 交互流程的完整性
- 社区知识与官方知识的边界管理

## 四、核心功能说明
### 1. 校园智能问答
用户可以直接输入自然语言问题，例如：

- 转专业后课程补退选怎么操作
- 奖学金材料需要准备什么
- 宿舍报修在哪里提交

系统会基于：
- 官方知识库
- 社区知识条目
- 大模型生成能力

输出结构化回答，并尽量附上来源信息。

### 2. RAG 检索增强
系统在大模型生成前会先进行知识检索。当前知识来源主要包括：

- 学校通知知识库
- 结构化附件索引
- 审核通过的社区知识条目

检索命中后，相关内容会作为上下文拼接到模型输入中，从而提升回答贴合度。

### 3. 数字人展示
首页采用数字人问答布局，支持：

- 首次进入播放欢迎视频
- 问答阶段切换到讲解视频
- 后端 TTS 播报
- 欢迎、待机、讲解三种状态切换

### 4. 学生社区
学生社区支持：

- 发布帖子
- 回复帖子
- 浏览帖子详情
- 展示已审核内容

社区内容不会直接进入主知识库，而是经过筛选、审核和知识生成后，才可作为辅助知识来源使用。

### 5. 管理员审核
管理员页支持：

- 审核帖子
- 审核回复
- 审核社区知识
- 查看候选帖子
- 生成社区知识条目

管理员只处理系统筛选后的高价值内容，从而减少人工工作量。

### 6. 登录注册
用户认证体系支持：

- 普通用户邮箱验证码注册
- 邮箱作为普通用户唯一标识
- 管理员账号密码登录
- 注册通知邮件发送

## 六、系统业务闭环
本项目目前已经形成了完整的业务闭环：

1. 用户登录系统
2. 用户在首页提问
3. 系统检索知识库并生成回答
4. 数字人展示讲解内容
5. 用户可查看来源信息
6. 学生可在社区补充经验内容
7. 管理员审核优质帖子与回复
8. 社区知识生成并进入审核流程
9. 审核通过后的社区知识反哺问答系统

## 七、项目亮点
### 1. 知识可追溯
回答不仅提供结果，还尽量附带来源依据，降低“黑盒回答”风险。

### 2. 社区与问答联动
学生社区不只是互动模块，还可作为辅助知识沉淀来源，为问答系统提供经验型补充。

### 3. 数字人交互体验
首页采用数字人展示方式，提升项目展示感和交互效果。

### 4. 角色化后台管理
管理员具备审核、筛选、知识沉淀能力，保证系统内容质量。

### 5. 适合比赛与展示
系统具备较完整的功能链路，适合用于课程设计、项目答辩、创新创业比赛与展示场景。

## 八、适用场景
- 校园智能问答平台
- 数字校园服务助手
- AI + 前端综合项目展示
- 高校智能服务系统答辩演示
- 智能体/RAG/数字人方向课程项目

## 九、后续可拓展方向
后续可以继续扩展：

- 更精细的知识库治理
- 社区知识质量评分机制
- 更自然的数字人口播与动作联动
- 小程序与移动端适配
- 更细粒度的权限体系
- 更强的知识统计与运营分析能力

## 十、总结
校园智能问答平台围绕“校园问答 + 数字人展示 + 社区知识沉淀 + 管理员审核”的整体思路，实现了一个具有完整业务闭环的校园智能服务系统。它不仅具备大模型问答能力，还通过知识库检索、社区知识筛选和管理员审核机制提升了回答质量与系统可信度，具有较好的展示价值和拓展潜力。
